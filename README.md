# FitBook — сервіс бронювання спортивних занять

## Опис ідеї
FitBook — веб-застосунок для пошуку та бронювання спортивних занять (йога, пілатес, crossfit, танці тощо).
Користувач бачить розклад, переглядає деталі заняття, додає в улюблені, бронює місце та керує своїми бронюваннями.

Проєкт розробляється як навчальний сервіс з повним циклом інженерних практик:
- оформлення вимог та структури
- інтерактивний прототип
- інтеграція з БД та віддаленими джерелами даних
- тестування (unit/integration/e2e)
- CI/CD і розгортання
- перформанс-аналіз і оптимізація
- винесення критичного модуля в окрему бібліотеку та перепис на TypeScript

## Ролі користувачів
- **USER**: перегляд розкладу, фільтрація, перегляд деталей, улюблені, бронювання, перегляд “Мої бронювання”.
- **ADMIN**: керування заняттями/розкладом (CRUD), керування локаціями/тренерами (за потреби).

## Основні сценарії (MVP)
1. Перегляд розкладу занять (список + фільтри).
2. Перегляд деталей заняття.
3. Додавання заняття в “Улюблені”.
4. Створення бронювання.
5. Перегляд “Мої бронювання”.

## Архітектура (високий рівень)
Монорепозиторій (pnpm workspaces):
- `apps/web` — Frontend (Next.js + TypeScript)
- `apps/api` — Backend (NestJS + TypeScript)
- `packages/shared` — спільні типи/DTO/валідації між web і api

## Технології
- TypeScript
- Next.js (frontend)
- NestJS (backend)
- Turborepo (monorepo pipelines)
- ESLint + Prettier (стиль коду)
- Husky + lint-staged (git hooks)
- commitlint + conventional commits (стиль комітів)

## Як запустити локально
> Потрібен Node.js (рекомендовано 20+) і pnpm

Встановити залежності:
```bash
pnpm install
```

Запуск у dev режимі:
```bash
pnpm dev
```

Форматування:
```bash
pnpm format
```

Перевірки:
```bash
pnpm lint
pnpm test
pnpm build
```

## План лабораторних
- **ЛР0:** вибір ідеї, репозиторій, README.
- **ЛР1:** код-стайл, prettier, lint, git hooks (commit/push), стаб-тести і збірка.
- **ЛР2:** модулі + діаграми (компонентів/ER) + опис оновлення даних у ключових сценаріях.
- **ЛР3:** інтерактивний прототип зі статичними даними.
- **ЛР4:** інтеграція з БД та заміна моків на реальні дані.
- **ЛР5:** unit/integration/e2e + мутаційне тестування та репорт.
- **ЛР6:** CI/CD, деплой на staging, сервіс доступний з інтернету.
- **ЛР7:** перформанс-аналіз (Lighthouse/навантаження/профілювання/БД).
- **ЛР8:** винесення bottleneck-модуля в бібліотеку + рефакторинг + перепис на TS.
